## 2. 已冻结的核心决策

### 2.1 业务方向

- 不是普通学生管理系统或图书管理系统
- 当前第一版聚焦“AI 内容精选与发布系统”
- 长期要扩展为“AI 信息聚合与精选平台”

### 2.2 技术路线

- 前台门户：Vue 3 + Vite + Vue Router + Pinia
- 后台管理：Vue 3 + Vite + Vue Router + Pinia + Element Plus
- 后端：Spring Boot 3 + MyBatis-Plus
- 数据库：MySQL 8
- 架构方式：前后端分离

### 2.3 项目边界

- 第一版不做自动采集
- 第一版不做复杂推荐算法
- 第一版不做多角色权限
- 第一版采用“无登录、单管理员演示模式”
- 第一版重点先把内容平台主流程做完整

- 后台首页已升级为内容运营控制台，包含关键指标卡、分布进度条、运营信号、近期内容和外部引用动态
- 本阶段不引入图表库，使用 CSS 轻量可视化，后台构建仍无 500KB chunk 警告
- 已真实验证接口返回统计数据，当前内容总数为 5，外部引用数为 4

- 后端标签筛选已改为查询前筛选，避免分页后过滤导致总数不准确

- 前台结果反馈已展示排序方式、筛选条件数量和结果数量
- 本阶段已验证热门排序、标签筛选、首页标签返回、前台构建、后台构建和后端编译

### AI 来源整理工作台增强后的当前可运行程度

- 已新增 6 类来源模板：结构化榜单源、官方动态源、GitHub 项目源、论文研究源、社区实践源、人工录入源
- AI 返回的标签建议会按名称匹配已有标签，匹配成功自动选中，未匹配内容只作为建议展示
- AI 结果支持两种确认路径：直接创建草稿，或回填到普通内容编辑表单继续人工修改
- 后端已优化百炼错误提示，包括未配置 API Key、输入过短、服务暂时无法访问、返回格式异常
- 本阶段仍不做爬虫、不做百炼联网搜索、不自动创建标签、不自动发布内容
- 本阶段已验证后端编译、前台构建、后台构建均通过，后台仍无 500KB chunk 警告

### 后台 API 设置与 GitHub API 接入后的当前可运行程度

- 密钥只保存在后端运行内存中，不落库、不回显完整密钥，后端重启后失效
- API 设置状态会显示已启用/未启用、掩码后缀、来源位置和更新时间
- 百炼调用已改为优先读取网页运行时配置，未配置时回退环境变量 `DASHSCOPE_API_KEY`

- 后台 GitHub 导入弹窗已支持真实仓库查询和关键词搜索，结果可回填导入表单
- 本阶段不新增数据库表、不做定时采集、不自动发布内容
