# 项目阶段复盘与问题清单

> 复盘日期：2026-04-29  
> 项目名称：AI前沿情报站  
> 当前定位：可展示、可答辩、可继续扩展的 AI 信息聚合与精选平台第一版

## 1. 当前项目已经做到什么程度

这个项目已经从最初的工程骨架，推进成一个完整的前后端分离 Web 系统。

当前已经具备：

- Vue 3 前台门户
- Vue 3 后台管理端
- Spring Boot 3 后端接口
- MySQL 数据库
- 内容、分类、标签、来源的基础管理
- 前台首页、内容广场、内容详情、项目说明页
- 后台内容管理、分类管理、标签管理、来源管理、Dashboard
- 真实数据库联调
- 统一返回结构、分页、异常处理
- Markdown 正文渲染
- 门户浅色 / 深色双主题
- 后台 GitHub-like 管理界面
- 内容录入实时预览
- 扩展字段模板
- 外部引用关系展示

从答辩角度看，它已经不是普通 CRUD 作业，而是一个有明确业务方向、有数据模型、有前后台分工、有未来扩展路径的内容平台。

## 6. 当前可能存在的问题

### 问题 1：还没有登录和权限

当前采用单管理员演示模式，适合课程第一版，但如果要上线就需要登录、权限和接口鉴权。

建议优先级：中。

### 问题 2：还没有真正自动采集

当前只做了数据源结构准备，没有实际调用 GitHub、arXiv 或官方博客 API。

建议优先级：中高。

### 问题 3：后台构建包体偏大

后台整包引入 Element Plus，构建时会提示 chunk 偏大。

建议优先级：中。

### 问题 4：Markdown 内容安全策略还不够完整

前台详情使用 `marked` 渲染 Markdown。当前数据来自后台人工录入，风险可控；如果未来接入外部来源，需要加入 HTML 清洗策略。

建议优先级：高，尤其在接外部采集前。

### 问题 5：后台没有深色模式

前台已经支持双主题，后台目前是浅色 GitHub-like 风格。

建议优先级：低到中。

### 问题 6：外部引用还缺少独立管理入口

阶段 12 已能展示外部引用，但还需要进一步完善手动新增、编辑和删除入口。

建议优先级：高。
