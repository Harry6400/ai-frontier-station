# 项目全景审计与长期执行计划

> 审计时间：2026-05-19
> 范围：后端(70+ Java文件) + 前台门户(11个视图) + 后台管理(7个视图) + 数据库
> 总计发现：65个问题（Critical 5 / High 9 / Medium 24 / Low 27）

---

## 一、架构级问题（必须解决）

### 1.1 CandidateAdminController 违反分层架构 [CRITICAL]
- **位置**：`backend/controller/admin/CandidateAdminController.java`
- **问题**：300行代码直接注入 Mapper，包含 AI 提示词构建、REST 调用、JSON 解析、内容映射等全部业务逻辑
- **影响**：不可测试、不可维护、无法复用
- **修复**：抽取 `CandidateService` 接口 + `CandidateServiceImpl`，Controller 只做委托
- **工期**：1天

### 1.2 硬编码分类/来源 ID [HIGH]
- **位置**：`CandidateAdminController.mapCategoryId()` / `mapSourceId()`
- **问题**：`return 7L`、`return 1L` 等硬编码，数据库重置后会指向错误记录
- **修复**：按 slug/name 查询，如 `categoryMapper.selectOne(...eq(slug, "ai-news"))`
- **工期**：0.5天

### 1.3 无 CORS 配置 [CRITICAL]
- **位置**：`backend/config/SecurityConfig.java`
- **问题**：前后端分离部署时跨域请求会被浏览器拦截
- **修复**：在 SecurityConfig 中添加 CorsConfigurationSource Bean
- **工期**：0.5天

### 1.4 前端无 404 路由 [MEDIUM]
- **位置**：`frontend-portal/src/router/index.js`
- **问题**：无效 URL 显示空白页
- **修复**：添加 `{ path: '/:pathMatch(.*)*', component: NotFoundView }`
- **工期**：0.5天

## 二、数据管道问题（影响内容质量）

### 2.1 Arena 板块 100% 硬编码数据 [HIGH]

### 2.2 产品动态板块部分硬编码 [MEDIUM]

- **问题**：AI周报摘要、产品列表、类型列表是硬编码
- **修复**：全部改为从 API 获取
- **工期**：0.5天

### 2.3 前台板块无分页 [MEDIUM]
- **位置**：NewsView / PapersView / GithubView / ToolsView
- **问题**：一次性获取 50 条，无分页控件，数据量大时性能差
- **修复**：添加分页组件
- **工期**：1天

### 2.4 前台无加载/错误/空状态统一处理 [MEDIUM]
- **位置**：NewsView / PapersView / GithubView / ToolsView / CompanyView
- **问题**：API 错误只 console.error，无用户提示；空数据无提示
- **修复**：抽取统一的 Loading / Error / Empty 组件
- **工期**：1天

### 2.5 采集管道无数据质量校验 [MEDIUM]

- **工期**：0.5天

## 三、安全问题（生产化前必须修复）

### 3.1 登录无暴力破解防护 [HIGH]
- **位置**：`AuthController` + `SecurityConfig`
- **问题**：无限次登录尝试，无锁定机制
- **修复**：添加速率限制或失败次数锁定
- **工期**：0.5天

### 3.2 JWT 登出未失效 [HIGH]
- **位置**：`AuthController.logout()`
- **问题**：登出后 token 仍有效（24h），被盗 token 可继续使用
- **修复**：实现 token 黑名单（Redis 或内存+TTL）
- **工期**：0.5天

### 3.3 全局异常处理暴露内部信息 [HIGH]
- **位置**：`GlobalExceptionHandler.java` 第42行
- **问题**：`ex.getMessage()` 直接返回给客户端，可能泄露 SQL 错误、类名
- **修复**：通用异常返回 "服务器内部错误"，详细信息只写日志
- **工期**：0.5天

### 3.4 分页参数无校验 [MEDIUM]
- **位置**：`ContentQueryRequest` / `PortalContentQueryRequest`
- **问题**：`pageSize=999999` 可以dump整个数据库
- **修复**：添加 `@Min(1) @Max(100)` 注解
- **工期**：0.5天

### 3.5 登录页暴露默认账号 [LOW]
- **位置**：`LoginView.vue` 第90行
- **问题**：显示 "默认账号：admin / admin123"
- **修复**：生产环境隐藏
- **工期**：0.5天

- **问题**：单文件过大，不可维护
- **修复**：拆分为 ContentListPanel / ContentFormDialog / GitHubImportDialog / ArxivImportDialog 等子组件

- **位置**：`ContentServiceImpl.toListItemVO()`

### 4.5 前台 axios 响应处理不一致 [MEDIUM]

- **问题**：有的用 `res.data?.records`，有的用 `res.data?.data`，防御性编码
- **修复**：统一 http.js 拦截器返回格式，所有 View 使用一致的解析方式
- **工期**：0.5天

### 4.6 死代码 [LOW]

  - `AboutView.vue` — 无路由引用

  - `ContentManageView.vue` 中 `ArrowDown` 导入未使用
- **修复**：删除
- **工期**：0.5天

### 4.7 LoginResponse 放在错误包中 [LOW]
- **位置**：`dto/request/LoginResponse.java`
- **修复**：移到 `vo/` 或 `dto/response/`
- **工期**：0.5天

## 五、数据库问题

### 5.1 无迁移策略 [MEDIUM]
- **问题**：schema.sql 用 `CREATE TABLE IF NOT EXISTS`，修改已有表需要手动 SQL
- **修复**：引入简单迁移表或使用 Flyway/Liquibase
- **工期**：1天

### 5.2 外键无 CASCADE [MEDIUM]
- **位置**：`ai_content_tag` / `ai_content_external_ref`
- **问题**：删除内容需手动先删关联记录
- **修复**：改为 `ON DELETE CASCADE` 或在 Service 层处理
- **工期**：0.5天

### 5.3 init-data.sql 硬编码 ID [MEDIUM]
- **问题**：INSERT 使用固定 category_id (3,2,4,5)，重跑会因 ID 不同而失败
- **修复**：使用子查询 `(SELECT id FROM ai_category WHERE slug=...)`
- **工期**：0.5天

### 5.4 缺少索引 [LOW]
- 缺少：`ai_content.title`（LIKE搜索）、`ai_content_external_ref.external_url`
- **修复**：添加索引
- **工期**：0.5天

### 5.5 冗余列 [LOW]
- **位置**：`ai_content_external_ref` 同时有 `synced_at` 和 `last_synced_at`
- **修复**：合并为一个
- **工期**：0.5天
