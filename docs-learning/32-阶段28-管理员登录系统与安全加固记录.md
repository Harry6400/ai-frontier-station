### API 接口

| Method | Path | 说明 | 认证要求 |
|--------|------|------|----------|
| POST | `/api/v1/auth/login` | 管理员登录 | 公开 |
| POST | `/api/v1/auth/logout` | 管理员登出 | 需认证 |
| GET | `/api/v1/auth/me` | 获取当前用户信息 | 需认证 |
| GET/POST/PUT/DELETE | `/api/v1/admin/**` | 管理接口 | 需认证 |
| GET | `/api/v1/portal/**` | 前台门户接口 | 公开 |

### 安全配置

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/v1/system/**").permitAll()
    .requestMatchers("/api/v1/portal/**").permitAll()
    .requestMatchers("/api/v1/auth/login").permitAll()
    .requestMatchers(HttpMethod.OPTIONS, "/**").permitAll()
    .requestMatchers("/api/v1/admin/**").authenticated()
    .anyRequest().authenticated()
)
```

## 答辩时可以怎么讲

### 安全架构

> 后台管理接口采用 Spring Security + JWT 实现无状态认证。前端登录后获取 JWT Token，存储在 localStorage 中。每次请求通过 Axios 拦截器自动注入 Authorization 头。后端通过 JwtAuthenticationFilter 验证 Token，有效的请求才能访问管理接口。

### 为什么选择 JWT

> JWT 是无状态的，不需要服务端保存 session，适合前后端分离架构。Token 中包含用户信息和过期时间，通过 HS256 签名保证不可篡改。前端可以方便地存储和传递 Token，后端可以轻松验证。

### 安全分层

> 系统分为三层安全防护：
> 1. **URL 级别**：Spring Security 配置哪些 URL 需要认证
> 2. **请求级别**：JwtAuthenticationFilter 验证每个请求的 Token
> 3. **前端级别**：路由守卫防止未登录用户访问管理页面
