# 阶段 32：定时采集引擎 + 去审核 + AI 中文摘要 + 首页大改

> 日期：2026-05-17 ~ 2026-05-18
> 这是项目从"人工运营 CMS"转型为"个人 AI 信息雷达"的关键阶段。

## 做了什么

### 1. 定时采集引擎（Phase A）

- 后端加 `@EnableScheduling`，新建定时任务模块
- **ArxivFetchScheduler**：每天凌晨 3 点自动跑，8 个关键词（low-dose CT、CT denoising、medical image denoising 等），每个关键词间隔 5 秒防限流
- **HuggingFaceFetchScheduler**：每天凌晨 4 点自动跑，拉取 Daily Papers 后按 13 个关键词过滤（medical、imaging、CT、denoising 等）
- 去重机制：查 `ai_content_external_ref` 表的 `ref_type + external_id`
- 采集日志表 `ai_fetch_log`：记录每次采集的来源、关键词、抓到条数、导入条数、状态
- Dashboard 集成采集统计
- 手动触发 API：`POST /api/v1/admin/fetch/trigger/arxiv|huggingface`

### 2. 去掉审核机制

- 所有内容直接以 PUBLISHED 状态入库，不再有 DRAFT
- 已有 64 篇 DRAFT 内容全部 UPDATE 为 PUBLISHED
- 后台前端删除"转草稿"按钮、DRAFT 默认值、相关文案
- ContentServiceImpl.options() 去掉"草稿"选项

### 3. AI 中文摘要管线（Phase B）

- 新建 PaperSummaryService，调 MiMo API 生成中文摘要
- Prompt 要求：中文撰写、面向本科生水平、包含核心贡献/方法简述/关联度评分/推荐理由
- 新论文入库后自动生成中文摘要（ArxivFetchScheduler 和 HuggingFaceFetchScheduler 集成）
- 批量摘要 API：`POST /api/v1/admin/fetch/trigger/summary`（对已有内容一次性跑完）

### 4. 前台首页大改

- 删除「栏目地图」和「这版首页为什么更像系统」两个模块
- 首页新增「板块导航」区域，3×2 网格排列 6 个入口卡片：
  - 论文速递 → `/contents?contentType=paper`
  - GitHub 开源项目 → `/contents?contentType=project`
  - AI 公司动态 → `/contents?contentType=company_update`
  - AI 资讯 → `/contents?contentType=news`
  - Arena 模型评测 → `/contents?contentType=arena`（新增类型，暂无内容）
  - 工具与最佳实践 → `/contents?contentType=practice`
- 导航栏更新：首页 | 论文 | GitHub | AI资讯 | 公司动态 | 工具实践
- ContentListView 加 Arena 模型评测类型选项

### 5. 新增 contentType：arena

- 后端 options() 加 `new OptionVO("Arena 模型评测", "arena")`
- 前端 typeOptions 和 portalStore 加对应选项
- 暂无内容，后续接 Arena 网站 API

## 踩的坑

1. **FetchLog 实体字段名不匹配**：子 agent 用了 fetchedCount/importedCount，但数据库是 papers_found/papers_imported。修复实体类。
2. **created_at 不能为 NULL**：项目没配 MetaObjectHandler，需要手动 setCreatedAt。
3. **summary 列太短**：VARCHAR(500) 装不下论文摘要，改为 TEXT。
4. **author_name 列太短**：VARCHAR(100) 装不下多作者拼接，改为 VARCHAR(500)。
5. **external_ref 缺列**：子 agent 加了 GitHub 同步字段但没跑 migration SQL，手动 ALTER TABLE。
6. **arXiv 限流**：3 秒间隔不够，改为 5 秒。凌晨 3 点正式跑时不会有这个问题。
7. **Spring Boot 不自动加载 .env**：需要手动 export 环境变量，或加 dotenv 依赖。
