# 阶段 13：后台外部引用管理入口记录

## 1. 本阶段目标

阶段 12 已经完成外部引用关系展示，但外部引用仍然主要来自初始化数据。本阶段进一步补齐后台维护入口，让管理员可以在内容编辑弹窗里维护外部引用。

本阶段完成后，`ai_content_external_ref` 不再只是“预留表”，而是进入后台内容运营流程。

## 2. 本阶段涉及的技术点

- Spring Boot REST 接口
- MyBatis-Plus 新增、修改、删除
- Vue 3 表单状态管理
- Element Plus Dialog、Form、DatePicker
- 外部引用与内容的父子关系校验
- 删除前确认

## 3. 当前实现范围

本阶段实现：

- 新增外部引用
- 编辑外部引用
- 删除外部引用
- 在后台内容编辑弹窗中展示外部引用
- 外部引用表单校验 URL 和 JSON

本阶段不实现：

- 自动采集
- 批量同步
- GitHub API 调用
- arXiv API 调用
- 定时任务

## 4. 后端如何设计

后端新增请求 DTO：

```text
ContentExternalRefSaveRequest
```

包含：

- `refType`
- `externalId`
- `externalUrl`
- `rawPayloadJson`
- `syncedAt`

`ContentService` 新增方法：

- `createExternalRef`
- `updateExternalRef`
- `deleteExternalRef`

`ContentAdminController` 新增接口：

```text
POST   /api/v1/admin/contents/{id}/external-refs
PUT    /api/v1/admin/contents/{id}/external-refs/{refId}
DELETE /api/v1/admin/contents/{id}/external-refs/{refId}
```

## 5. 为什么接口挂在内容下面

外部引用不是独立内容，它必须属于某一条内容。

所以接口设计成：

```text
/contents/{id}/external-refs
```

这样能表达父子关系，也方便后端校验：

> refId 对应的外部引用，必须属于当前 contentId。

答辩时可以这样说：

> 外部引用是内容的附属资源，所以我把它设计成内容下面的子资源接口，而不是单独暴露成完全独立的模块。

## 6. 后台怎么使用

在后台内容管理页：

1. 点击某条内容的“编辑”
2. 右侧预览区可以看到“外部引用记录”
3. 点击“新增”可以创建外部引用
4. 每条外部引用支持打开、编辑、删除
5. 删除前会弹出确认框

## 7. 当前安全边界

删除外部引用属于数据删除操作，所以页面会弹出确认框。

当前没有自动调用删除操作，只有用户在后台主动点击删除并确认后才会删除。

## 8. 老师可能会怎么问

### 问：为什么外部引用要支持维护？

可以回答：

> 因为未来采集模块同步来的内容，需要记录它对应的外部原始记录。即使第一版没有自动采集，后台也应该能人工维护这些映射关系，方便演示和扩展。

### 问：为什么修改外部引用时要校验 contentId？

可以回答：

> 因为外部引用是内容的子资源。校验 contentId 可以避免把 A 内容的引用误改到 B 内容上。

### 问：这个功能和来源管理有什么区别？

可以回答：

> 来源管理维护的是数据源，例如 GitHub Trending、arXiv；外部引用维护的是某条内容对应的外部原始记录，例如某个 GitHub 仓库或某篇论文。
