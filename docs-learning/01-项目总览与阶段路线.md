# 项目总览与阶段路线

## 1. 这个项目到底是什么

AI前沿情报站的第一版，本质上是一个“AI 内容精选与发布系统”。

它要同时承担两个角色：

- 课程作业中的完整 Web 系统
- 未来 AI 信息聚合平台的基础底座

所以它不能只是几个页面拼接，也不能只为了短期演示把结构写死。

## 2. 为什么选择这个方向

这个方向相比普通管理系统有几个优势：

- 更符合 AI 时代背景，题目更新
- 既能展示内容管理，也能展示前台内容门户
- 数据库设计、接口设计、扩展设计都比较好讲
- 后续能自然扩展到 GitHub、论文、公司动态、技术实践等真实方向

## 3. 第一版系统目标

第一版先完成以下闭环：

- 后台录入内容
- 后台管理分类、标签、来源
- 后台控制发布状态
- 前台展示精选内容
- 前台支持列表、详情、搜索与筛选

## 4. 为什么第一版不直接做自动采集

不是因为自动采集不重要，而是因为：

- 自动采集会明显拉高项目复杂度
- 当前阶段更重要的是先把“内容平台主干”做稳
- 先把内容模型、来源模型、发布模型设计好，后面再接采集模块更合理

这也是答辩时很重要的一点：

> 先完成稳定的业务核心，再做能力扩展，而不是一开始堆很多复杂功能但系统主干不稳。

## 5. 阶段路线

### 阶段 1：工程骨架与文档初始化

- 建目录
- 建教学文档
- 建后端骨架
- 建前端骨架
- 建数据库初稿

### 阶段 2：后端基础能力

- 接入数据库
- 建立内容、分类、标签、来源表对应的实体与接口
- 统一返回结构
- 统一异常处理

### 阶段 3：后台管理功能

- 内容新增、编辑、删除、发布
- 分类管理
- 标签管理
- 来源管理

### 阶段 4：前台门户设计与实现

- 用 `huashu-design` 做首页与内容卡片方向稿
- 实现门户首页
- 实现列表页、详情页、筛选区

### 阶段 5：联调与展示优化

- 前后端联调
- 补充假数据
- 完善页面状态
- 完善答辩文档

### 阶段 6：预留长期扩展点

- GitHub 项目接入预留
- 论文信息接入预留
- 公司动态接入预留
- 推荐、热度、专题、收藏等模块预留
