### 1.4 新建候选审核页面
文件：`frontend-admin/src/views/CandidateReviewView.vue`

布局：
```
┌─────────────────────────────────────────────────────┐
│ 顶部工具栏                                           │
│ [来源Tab: 全部|GitHub|新闻|产品动态|Arena|工具实践]   │
│ [状态Tab: 待审核|已批准|已拒绝|已导入]               │
│ [触发采集按钮: 采集GitHub / 采集新闻 / ...]          │
├─────────────────────────────────────────────────────┤
│ 候选列表（左侧，60%宽度）                            │
│ ┌─────────────────────────────────────────────────┐ │
│ │ 标题 | 来源 | 状态 | 时间 | 操作[批准][拒绝]    │ │
│ │ ...                                             │ │
│ └─────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────┤
│ 候选详情（选中后展开或右侧40%）                      │
│ ┌──────────────────────┬──────────────────────────┐ │
│ │ 原始数据             │ AI处理结果               │ │
│ │ (raw_content)        │ (ai_summary + ai_body)   │ │
│ │ 可编辑               │ 可编辑                   │ │
│ └──────────────────────┴──────────────────────────┘ │
│ [保存编辑] [批准并导入] [拒绝]                       │
└─────────────────────────────────────────────────────┘
```

功能：
- 顶部来源Tab切换过滤（对应source_type字段）
- 状态Tab切换（pending/approved/rejected/imported）
- 触发采集按钮（调用POST /admin/candidates/fetch/{sourceType}）
- 候选列表显示：标题、来源、状态、发布时间
- 点击候选展开详情面板
- 详情面板分两列：原始数据（可编辑rawContent）、AI处理结果（可编辑aiSummary/aiBody）
- 操作：保存编辑（PUT）、批准并导入（POST approve）、拒绝（POST reject）
- 批准时可选择覆盖标题/摘要/正文

复用样式：
- 参考ContentManageView的列表样式和编辑弹窗样式
- 使用Element Plus的el-tabs、el-table、el-button等组件

## Part 2：4个采集服务

### 2.1 NewsRssService（RSS新闻采集）
文件：
- `backend/service/NewsRssService.java`（接口）
- `backend/service/impl/NewsRssServiceImpl.java`（实现）
- `backend/scheduler/NewsFetchScheduler.java`（定时任务）

数据源：
- TechCrunch RSS: `https://techcrunch.com/feed/`
- The Verge AI: `https://www.theverge.com/rss/ai-artificial-intelligence/index.xml`
- Hacker News API: `https://hacker-news.firebaseio.com/v0/topstories.json` + `https://hacker-news.firebaseio.com/v0/item/{id}.json`

实现方式：
- 使用RestClient获取RSS XML
- 用javax.xml.parsers.DocumentBuilder解析RSS/Atom格式
- 提取：title, link, description, pubDate, author
- 对HN API：获取top 30 stories，过滤AI相关内容（标题包含ai/llm/gpt/model等关键词）
- 存入ai_content_candidate（source_type='news'）
- 跳过已存在的（按url去重）

定时：每天08:00和20:00

### 2.2 ProductUpdateService（产品动态采集）
文件：
- `backend/service/ProductUpdateService.java`
- `backend/service/impl/ProductUpdateServiceImpl.java`
- `backend/scheduler/ProductFetchScheduler.java`

数据源：
- OpenAI Blog RSS: `https://openai.com/blog/rss.xml`（如果有的话，否则用API）
- Anthropic Blog: `https://www.anthropic.com/rss.xml`
- Google AI Blog: `https://blog.google/technology/ai/rss/`
- GitHub Releases: `GET /repos/{owner}/{repo}/releases`（for open-source tools like vllm, ollama, etc.）

实现方式：
- RestClient获取RSS/JSON
- 解析标题和内容
- 通过标题关键词推断产品归属（ChatGPT→OpenAI, Claude→Anthropic, Gemini→Google, DeepSeek→DeepSeek, Qwen→阿里云, Trae→字节跳动）
- metadata_json存{product, company, updateType}
- 存入ai_content_candidate（source_type='product'）
- 按url去重

定时：每天09:00

### 2.3 ArenaService（模型评测数据采集）
文件：
- `backend/service/ArenaService.java`
- `backend/service/impl/ArenaServiceImpl.java`
- `backend/scheduler/ArenaFetchScheduler.java`

数据源：
- LMSYS Chatbot Arena 排行榜页面：`https://chat.lmsys.org/`（需要抓取HTML）
- 如果有API端点优先用API

实现方式：
- RestClient获取Arena排行榜页面HTML
- 解析HTML提取排名数据（模型名、ELO分数、投票数）
- 如果页面是动态渲染（React/JS），可能需要用其他方式
- 备选方案：手动更新或使用已知的Arena API（如果存在）
- 存入ai_content_candidate（source_type='arena'）
- metadata_json存{model, elo, rank, votes}

定时：每天10:00

**风险**：Arena页面可能是SPA动态渲染，RestClient获取的HTML可能不含数据。需要实际测试。

### 2.4 ToolsRssService（工具实践采集）
文件：
- `backend/service/ToolsRssService.java`
- `backend/service/impl/ToolsRssServiceImpl.java`
- `backend/scheduler/ToolsFetchScheduler.java`

数据源：
- Reddit r/MachineLearning JSON: `https://www.reddit.com/r/MachineLearning/hot.json`
- Reddit r/LocalLLaMA JSON: `https://www.reddit.com/r/LocalLLaMA/hot.json`
- Hacker News API（与News共用，但过滤工具/practice相关内容）
- Product Hunt（如果有API）

实现方式：
- RestClient获取Reddit JSON（加User-Agent头，Reddit要求）
- 解析帖子标题、内容、作者、upvotes、链接
- 过滤AI工具相关内容（标题包含tool/workflow/setup/deploy/benchmark/compare等）
- 存入ai_content_candidate（source_type='tools'）
- 按url去重

定时：每天12:00
