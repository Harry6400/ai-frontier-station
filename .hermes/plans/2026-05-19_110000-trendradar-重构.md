# TrendRadar 重构计划

## 核心理念

从"抓更多数据"转向"精准发现值得展示的AI趋势"。
系统定位：AI Trend Intelligence Platform，不是RSS新闻站。

## 当前问题

1. GitHub采集按total stars排序 → 永远是transformers/llama霸榜
2. 没有趋势评分 → 无法发现"最近爆火但总star不高"的项目
3. AI只做简单摘要 → 没有标签/难度/受众/学习路线
4. 没有时间衰减 → 旧内容永远排在前面
5. 新闻/工具采集质量低 → RSS堆砌，没有筛选

## 重构范围（分3期）

#### 1.2 GitHub Trend Score 算法

```
trend_score = (
    star_growth_7d_weight   * 0.35 +   -- 7天star增长（核心信号）
    fork_growth_weight      * 0.10 +   -- fork增长
    recency_weight          * 0.15 +   -- 最近更新时间
    release_frequency       * 0.10 +   -- 最近发布频率
    discussion_activity     * 0.10 +   -- issue/PR活跃度
    source_confidence       * 0.05 +   -- 来源可信度
    ai_relevance            * 0.10 +   -- AI相关性（关键词匹配）
    time_decay              * -0.05    -- 时间衰减（越老扣分越多）
)
```

**实现方式：**
- GitHub Search API: `q=stars:>100 pushed:>{7天前日期} topic:ai OR topic:machine-learning OR topic:llm`
- 按 `stars` 排序获取近期活跃项目
- 对比7天前的star数计算增长量（用GitHub API的star history或缓存上次数据）
- 计算最终trend_score

**去重策略：**
- 按 `external_id`（owner/repo）去重
- 每次采集更新已有记录的star数和trend_score
- 不重复创建新记录

