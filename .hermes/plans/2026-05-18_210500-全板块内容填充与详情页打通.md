# 全板块内容填充 + 详情页打通计划

## 目标
让6个板块都有可点击、可阅读的真实内容，不再只是mock数据摆设。

## 现状

| 板块 | 数据库内容 | 前端数据 | 能否点进详情 |
|------|-----------|---------|-------------|
| 论文 | 96条真实数据 | 已对接API | ✅ 可以 |
| GitHub | 0条 | mock数据 | ❌ 不能 |
| AI资讯 | 0条 | mock数据 | ❌ 不能 |
| 产品动态 | 0条 | mock数据 | ❌ 不能 |
| 模型评测 | 0条 | mock数据(排名图) | ❌ 不能 |
| 工具实践 | 0条 | mock数据 | ❌ 不能 |

详情页 `ContentDetailView.vue` 已存在，支持渲染 title/summary/bodyMarkdown/sourceUrl/extraJson/externalRefs，但只有论文能用。

## 方案

### 核心思路：给每个板块插入种子数据，前端对接API，点击进入统一详情页

不做爬虫/定时任务（那是后续阶段），先用手工插入的高质量种子内容让网站"活起来"。

---

## 执行步骤

### 步骤1：为每个板块插入种子内容（SQL）

每个板块插入3-5条真实感内容，内容来源：
- **GitHub** (content_type=project, category_id=?): 从GitHub Trending抓取的真实项目信息，写成中文总结
- **AI资讯** (content_type=news, category_id=6): 近期真实AI新闻
- **产品动态** (content_type=company_update, category_id=7): 近期真实产品更新
- **模型评测** (content_type=arena, category_id=8): Arena排名解读
- **工具实践** (content_type=practice, category_id=9): 真实工具分享

每条内容包含：
- title: 中文标题
- slug: 英文标识
- content_type: 对应类型
- sub_category: 可选子分类
- summary: 100字以内摘要
- body_markdown: 完整正文（Markdown格式，300-800字）
- category_id: 对应分类
- source_id: 对应来源
- source_url: 原始链接
- author_name: 来源作者
- publish_status: PUBLISHED
- published_at: 近期时间
- extra_json: 扩展信息

**文件变更**: 新建 `database/seed-sections.sql`

### 步骤2：更新前端6个板块对接API

每个板块的改动模式相同：
1. 删除mock数据ref
2. 导入 `getContentByType` from `../api/portal`
3. `onMounted` 时调API获取数据
4. 列表项用 `RouterLink :to="'/contents/' + item.id"` 包裹，点击进入详情页

**文件变更**:
- `frontend-portal/src/views/GithubView.vue`
- `frontend-portal/src/views/NewsView.vue`
- `frontend-portal/src/views/CompanyView.vue`
- `frontend-portal/src/views/ToolsView.vue`
- `frontend-portal/src/views/ArenaView.vue`（排名图保留mock，下方补充详情内容列表）

### 步骤3：优化 ContentDetailView 适配所有类型

当前详情页对论文效果好，但其他类型需要调整：
- 去掉"阅读时长"显示（已从编辑器删除）
- 子分类标签显示（paper显示sub_category中文名）
- GitHub项目显示star/fork等extra_json信息
- 产品动态显示产品名和更新类型
- AI资讯显示来源地区
- 工具实践显示平台和作者信息

**文件变更**: `frontend-portal/src/views/ContentDetailView.vue`

### 步骤4：补充 content.js 类型映射

当前 CONTENT_TYPE_META 缺少 arena 类型，需要补充：
```javascript
arena: { label: '模型评测', english: 'Model Arena' }
```

**文件变更**: `frontend-portal/src/utils/content.js`

## 验证步骤

1. 执行 `seed-sections.sql` 后检查: `SELECT content_type, COUNT(*) FROM ai_content GROUP BY content_type`
2. 后端编译: `cd backend && mvn -q -DskipTests compile`
3. API验证: `curl localhost:8080/api/v1/portal/contents?contentType=news` 应返回5条
4. 前端构建: `cd frontend-portal && npm run build`
5. 浏览器验证: 每个板块的列表项可点击，进入详情页显示完整内容
