# AI前沿情报站 — 6板块数据对接与清理计划

## 目标
清理不适配新6板块结构的旧数据，重构分类体系，对接真实API，让6个板块都有真实数据支撑。

## 现状摸底

### 数据库现状
| 项目 | 数据 |
|------|------|
| 分类 | 5个: AI资讯(1), 论文速递(2), 热门项目(3), 公司动态(4), 技术实践(5) |
| 内容 | 104条: paper(96), project(3), practice(4), company_update(1) |
| 论文 | 96条，其中93条CT/医学影像相关，无子分类字段 |
| 来源 | 6个: GitHub Trending, arXiv, OpenAI Blog, DeepSeek News, 人工录入, HuggingFace |
| 外部引用 | 99条: arxiv_paper(45), huggingface_paper(51), github_repo(3) |

### 新6板块需求
1. **论文** — 3个子板块(3D CT去噪/医学影像/大模型) + AI周总结
2. **GitHub** — 每周Top10星标增长
3. **AI资讯** — 国际AI政策 + 行业实践
4. **产品动态** — 13个产品/公司的模型/API/IDE/价格更新
5. **模型评测** — Arena ELO排名
6. **工具实践** — 社交平台KOL分享

### 不适配的内容
- 96条论文中只有CT/医学影像，没有"大模型"子分类数据，且无paperVenue标记
- project(3条)和practice(4条)是早期手动录入的笼统文章，不适配新板块粒度
- company_update(1条)是旧格式对比文章，不适配"产品动态"板块
- 无news类型内容（AI资讯板块完全空）
- 无arena类型内容（模型评测板块完全空）
- content_type只有5种，需要新增 `news` 和 `arena`

## 执行计划

### 阶段1: 数据库清理与重构

**1.1 论文子分类**
- 给 `ai_content` 表加 `sub_category VARCHAR(50)` 字段
- 93条CT/医学影像论文: 根据标题关键词细分
  - 包含 "3D", "CT", "denois", "low-dose" → `sub_category = '3d_ct_denoising'`
  - 其余医学影像 → `sub_category = 'medical_imaging'`
- 3条非CT论文 → `sub_category = 'large_model'`（或根据标题判断）
- 后续新抓取的论文自动打子分类标签

**1.2 清理不适配内容**
- 删除 project(3条), practice(4条), company_update(1条) — 共8条
- 这些是早期手动录入的笼统文章，粒度和格式都不匹配新板块
- 同时清理对应的 ai_content_external_ref 和 ai_content_tag 记录

**1.3 更新分类表**
- 删除旧分类: 热门项目(3), 公司动态(4), 技术实践(5)
- 新增分类: AI资讯(6), 产品动态(7), 模型评测(8), 工具实践(9)
- 保留: 论文速递(2) — 但前端用sub_category做子tab切换
- 论文速递的 category_id 保持不变

**1.4 新增来源**
- 新增来源: Chatbot Arena (source_type=leaderboard)
- 新增来源: X/Twitter (source_type=social)
- 新增来源: Reddit (source_type=community)
- 新增来源: Hacker News (source_type=community)
- 新增来源: Product Hunt (source_type=community)

**1.5 SQL脚本**
```sql
-- 1.5.1 加sub_category字段
ALTER TABLE ai_content ADD COLUMN sub_category VARCHAR(50) DEFAULT NULL COMMENT '子分类' AFTER content_type;

-- 1.5.2 论文子分类标记
UPDATE ai_content SET sub_category = '3d_ct_denoising' 
WHERE content_type = 'paper' AND (title REGEXP '3D|CT|denois|low.dose|cone.beam' OR summary REGEXP 'CT去噪|低剂量|3D.*去噪');
UPDATE ai_content SET sub_category = 'medical_imaging' 
WHERE content_type = 'paper' AND sub_category IS NULL;

-- 1.5.3 删除不适配内容
DELETE FROM ai_content_tag WHERE content_id IN (SELECT id FROM ai_content WHERE content_type IN ('project', 'practice', 'company_update'));
DELETE FROM ai_content_external_ref WHERE content_id IN (SELECT id FROM ai_content WHERE content_type IN ('project', 'practice', 'company_update'));
DELETE FROM ai_content WHERE content_type IN ('project', 'practice', 'company_update');

-- 1.5.4 更新分类表
UPDATE ai_category SET name = '论文', slug = 'papers', description = 'arXiv + HuggingFace 论文速递' WHERE id = 2;
INSERT INTO ai_category (id, name, slug, description, sort_order) VALUES (6, 'AI资讯', 'news', '全球AI政策与行业实践', 3) ON DUPLICATE KEY UPDATE name='AI资讯';
INSERT INTO ai_category (id, name, slug, description, sort_order) VALUES (7, '产品动态', 'products', 'AI产品与公司动态', 4) ON DUPLICATE KEY UPDATE name='产品动态';
INSERT INTO ai_category (id, name, slug, description, sort_order) VALUES (8, '模型评测', 'arena', '模型评测排行榜', 5) ON DUPLICATE KEY UPDATE name='模型评测';
INSERT INTO ai_category (id, name, slug, description, sort_order) VALUES (9, '工具实践', 'tools', 'AI工具与实践分享', 6) ON DUPLICATE KEY UPDATE name='工具实践';
DELETE FROM ai_category WHERE id IN (1, 3, 4, 5);
```
